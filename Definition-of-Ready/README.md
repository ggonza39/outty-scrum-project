# Definition of Ready (DoR)

A Product Backlog Item (PBI) is considered "Ready" when it meets all of the following criteria:

## 1. Title
- The backlog item has a clear, concise, and descriptive title.

## 2. User Story Format
- The backlog item is written in the standard user story format:
  
  **As a [type of user], I want [some goal] so that [some reason].**

## 3. Additional Details
- The backlog item includes sufficient details to be understood by the development team.
- Acceptance criteria or clarifying notes are provided when necessary.
- Dependencies or assumptions are identified.

## 4. Estimated in Story Points
- The backlog item has been estimated using story points based on complexity, effort, and uncertainty.
- Estimation is performed relative to other backlog items.

Only backlog items that meet this Definition of Ready may be moved into a sprint.
